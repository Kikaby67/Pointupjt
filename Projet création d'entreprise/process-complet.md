# Process complet — Lancement business Pointu-PJT
## De zéro à la première vente

---

## PHASE 1 — Finalisation du produit (Semaines 1–3)

### 1.1 Fichier Streamer.bot
- [ ] Exporter le fichier `.sb` complet avec toutes les commandes RPG
- [ ] Tester sur un compte Twitch de test (streamchat simulé)
- [ ] Vérifier la connexion Discord (rôles, messages auto)
- [ ] Versionner le fichier (v1.0.0)

### 1.2 Contenu du pack
- [ ] Créer le PDF tuto pas-à-pas (avec captures d'écran)
  - Installation Streamer.bot
  - Import du fichier `.sb`
  - Configuration Discord (bot token, channel IDs)
  - Test en conditions réelles
  - Personnalisation (noms, lore, commandes custom)
- [ ] Créer un fichier `README.txt` inclus dans le zip

### 1.3 Documentation interne
- [ ] Écrire la FAQ (10–15 questions fréquentes)
- [ ] Créer un lexique des commandes disponibles

---

## PHASE 2 — Infrastructure business (Semaines 2–4)

### 2.1 Entité légale
- [ ] Choisir le statut : **Auto-entrepreneur** (recommandé pour démarrer)
  - Démarche sur autoentrepreneur.urssaf.fr
  - Code APE probable : 6201Z (Programmation informatique)
- [ ] Ouvrir un compte bancaire dédié (ou compte pro Shine/Qonto)
- [ ] Déclarer l'activité

### 2.2 Système de vente
- [ ] Créer un compte **Gumroad** (0€, commission ~10%) ou **Lemon Squeezy**
- [ ] Créer les 3 offres (Starter / Pro / Studio)
- [ ] Préparer les ZIPs de chaque pack
- [ ] Configurer la livraison automatique

### 2.3 Présence en ligne
- [ ] Créer un serveur Discord "Pointu Community" (support + annonces)
- [ ] Créer une page Linktree ou Carrd (vitrine rapide, gratuit)
  - Description du concept
  - Lien vers les packs
  - Lien Discord
- [ ] Créer un compte TikTok/Twitter/X dédié au projet

---

## PHASE 3 — Bêta test (Semaines 3–5)

### 3.1 Recrutement bêta-testeurs
- [ ] Contacter 3–5 streamers dans le réseau personnel
- [ ] Proposer le pack gratuitement en échange d'un feedback structuré
- [ ] Créer un formulaire Google Forms pour le retour bêta

### 3.2 Itération
- [ ] Analyser les retours (points bloquants, bugs, incompréhensions tuto)
- [ ] Corriger le fichier `.sb` si nécessaire
- [ ] Améliorer le tuto en fonction des retours
- [ ] Faire valider la v1.0 finale par au moins 1 bêta-testeur

### 3.3 Témoignages
- [ ] Demander un témoignage écrit ou vidéo à chaque bêta-testeur satisfait
- [ ] Capturer une session de jeu en stream pour démo vidéo

---

## PHASE 4 — Lancement (Semaine 5–6)

### 4.1 Contenu marketing
- [ ] Vidéo de démo courte (60–90 sec) : "Voilà ce que ça donne en stream"
- [ ] Carrousel ou post explicatif "Comment ça marche en 4 étapes"
- [ ] Post de lancement avec témoignage bêta-testeur

### 4.2 Distribution
- [ ] Publication sur TikTok / Twitter / X / Reddit (r/Twitch, r/livesportsstreaming)
- [ ] Partage dans les groupes Facebook streamers FR
- [ ] Contacter 10–20 streamers ciblés en DM (message personnalisé, pas de spam)
- [ ] Proposer un code promo lancement (-20%) valable 7 jours

### 4.3 Suivi lancement
- [ ] Surveiller les ventes quotidiennement pendant 2 semaines
- [ ] Répondre à chaque question/message dans les 24h
- [ ] Collecter les nouveaux avis clients

---

## PHASE 5 — Croissance (Mois 2–6)

### 5.1 Produit
- [ ] Publier une v1.1 avec correctifs et nouvelles commandes
- [ ] Développer 1–2 scénarios supplémentaires (Pack Pro)
- [ ] Réfléchir à une version anglophone

### 5.2 Acquisition
- [ ] Créer du contenu régulier (1x/semaine min.) sur les réseaux
- [ ] Programme d'affiliation : offrir 20% de commission à tout streamer qui recommande
- [ ] Partenariat avec 1 streamer moyen (500–2000 viewers) pour démo live

### 5.3 Abonnement (Phase 2)
- [ ] Évaluer si le modèle abonnement est pertinent (après 50+ clients)
- [ ] Si oui : configurer Lemon Squeezy ou Stripe Billing
- [ ] Migrer les clients existants avec tarif préférentiel

---

## PHASE 6 — Stabilisation (Mois 6–12)

- [ ] Automatiser le support (bot Discord FAQ)
- [ ] Créer une base de connaissance en ligne (Notion public ou GitBook)
- [ ] Évaluer l'embauche d'un support communautaire (bénévole ou rémunéré)
- [ ] Bilan comptable trimestriel
- [ ] Définir l'objectif Année 2 : scale FR ou pivot international

---

## Indicateurs de succès (KPIs)

| Indicateur | Objectif Mois 3 | Objectif Mois 12 |
|---|---|---|
| Packs vendus | 20 | 120 |
| CA cumulé | 700€ | 5 000€ |
| Membres Discord | 50 | 300 |
| Taux de satisfaction (bêta) | ≥ 80% | ≥ 90% |
| Streamers actifs utilisant le pack | 10 | 80 |

## Stack technique minimale

|---|---|---|

## Récapitulatif des délais

```
Semaine 1-3  : Finalisation produit + tuto
Semaine 2-4  : Création entité légale + infrastructure vente
Semaine 3-5  : Bêta test + itération
Semaine 5-6  : Lancement public
Mois 2-6     : Croissance + Pack Pro
Mois 6-12    : Stabilisation + Année 2
```

---

> **Principe directeur** : Vendre avant de sur-construire. Une vente réelle vaut 100 fonctionnalités de plus.
